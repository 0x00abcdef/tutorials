# 第四章 使用配置中心 doing

前面的章节中，我们把配置以yml文件的方式放到配置目录**conf**中，并通过**go-config**把其加载读到应用中。

Micro生态链中使用**go-config**管理配置，但是目前**go-config**定位是客户端插件，即是说它并不没有充当配置中心的能力，不过，它有接口配置使用其它如consul、etcd、k8s等具备kv存储能力的配置服务器。

通过与配置服务器配合，就构成了配置中心。
